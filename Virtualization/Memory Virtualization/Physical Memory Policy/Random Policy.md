### 무작위 선택
-----
1. 메모리 압박이 있을 때 페이지를 무작위로 선택하여 교체
2. FIFO와 유사한 성질을 가지고 있으며, 구현하기 쉽지만 블럭을 제대로 선택하려는 노력을 하지 않음
3. 예제

   - 무작위 선택 정책은 선택할 때 얼마나 운이 좋은지 (또는 운이 나쁜지)에 전적으로 의존
   - 무작위 선택 방식의 일반적인 성능을 알아보기 위해서는 실험을 수천 번 반복할 수 있음
   - 서로 다른 랜덤 시드를 가지는 실험을 만 번 수행했을 때 무작위 선택 방식의 히트 정도

   - 때로는 매우 안좋은 성능을 보이며, 2번의 히트 또는 그 이하를 얻음
   - 즉, 무작위 선택 방식 동작은 그때마다 달라짐

### Belady's Anomaly
-----
1. 캐시의 크기가 커지면서 캐시 히트율이 증가하는 것을 일반적으로 기대
2. 하지만 FIFO를 사용하는 경우 더 안 좋아지는데, 이 현상을 Belady's Anomaly
3. LRU와 같은 다른 정책은 이런 문제를 겪지 않음
   - LRU는 스택 특성(Stack Property)이라고 알려진 특성을 가짐
   - 이 특성을 가지고 있는 알고리즘들의 경우에는 캐시 크기가 N + 1로 증가하게 되면 자연스레 캐시 크기 N일 때 내용을 포함하게 됨
   - 그러므로 캐시 크기가 증가하면 히트율은 그대로 유지되거나 향상
   - 하지만, 여러 정책들 중 FIFO와 랜덤 정책은 이 스택 특성을 따르지 않고 변칙적인 행동을 보여줌
